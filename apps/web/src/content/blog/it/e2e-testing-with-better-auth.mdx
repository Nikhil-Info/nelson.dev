## TL;DR

Dovrai generare un token di sessione valido e salvarlo come cookie. Ciò consente ai tuoi test E2E di accedere a route autenticate senza login manuale. Questo metodo funziona sia per l'autenticazione basata su credenziali che per OAuth.

> Repository GitHub: [nelsonlaidev/e2e-testing-with-better-auth](https://github.com/nelsonlaidev/e2e-testing-with-better-auth)

## Prefazione

In questa guida utilizzeremo [Playwright](https://playwright.dev/) come nostro framework per i test E2E, ma i concetti possono essere applicati ad altri framework come [Cypress](https://www.cypress.io/).

Per semplicità, utilizzeremo l'esempio di autenticazione basata su credenziali. Tuttavia, gli stessi principi si applicano ai provider OAuth.

Utilizziamo anche un database SQLite a scopo dimostrativo. Configura le interazioni con il database **(ad esempio, nomi tabelle, tipi di colonne)** in base alla tua configurazione.

## Generazione del token di sessione

Per simulare una sessione autenticata, dobbiamo generare un token di sessione firmato utilizzando `BETTER_AUTH_SECRET`. Questo token verrà utilizzato nel cookie di sessione.

```ts
import crypto from 'node:crypto'

export const TEST_USER = {
  name: 'Utente Test',
  email: 'test@example.com',
  sessionToken: '00000000000000000000000000000000',
  accountId: '000'
}

const signature = crypto
  .createHmac('sha256', process.env.BETTER_AUTH_SECRET!)
  .update(TEST_USER.sessionToken)
  .digest('base64')
const signedValue = `${TEST_USER.sessionToken}.${signature}`
```

## Inserimento dei dati di test

Vogliamo mantenere il nostro utente di test statico per evitare di creare più utenti durante i test. Inserisci l'utente di test, l'account e la sessione nel database se non esistono già. Utilizziamo `0` come identificatore univoco per tutte le chiavi primarie per semplicità.

```ts
import { db } from '@/lib/db'

const TEST_UNIQUE_ID = '0'

const now = new Date().getTime()
const expiresAt = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000).getTime() // 7 giorni

const transaction = db.transaction(() => {
  db.prepare(
    `
      INSERT OR IGNORE INTO user (
        id, name, email, emailVerified, createdAt, updatedAt
      ) VALUES (?, ?, ?, ?, ?, ?)
    `
  ).run(TEST_UNIQUE_ID, TEST_USER.name, TEST_USER.email, 0, now, now)

  db.prepare(
    `
      INSERT OR IGNORE INTO account (
        id, accountId, providerId, userId, password, createdAt, updatedAt
      ) VALUES (?, ?, ?, ?, ?, ?, ?)
    `
  ).run(TEST_UNIQUE_ID, TEST_USER.accountId, 'credential', TEST_UNIQUE_ID, 'password_hash', now, now)

  db.prepare(
    `
      INSERT INTO session (
        id, token, userId, expiresAt, createdAt, updatedAt
      ) VALUES (?, ?, ?, ?, ?, ?)
      ON CONFLICT(token) DO UPDATE SET
      expiresAt = excluded.expiresAt,
      updatedAt = excluded.updatedAt
    `
  ).run(TEST_UNIQUE_ID, TEST_USER.sessionToken, TEST_UNIQUE_ID, expiresAt, now, now)
})

transaction()
```

## Salvataggio della sessione

Salva il token firmato come cookie in un file JSON per l'uso nei framework di test. Ricorda di codificare il valore del cookie con `encodeURIComponent{:.entity.name.function}`.

```ts
import fs from 'node:fs/promises'

const cookieObject = {
  name: 'better-auth.session_token',
  value: encodeURIComponent(signedValue), // [!code highlight]
  domain: 'localhost',
  path: '/',
  httpOnly: true,
  secure: false,
  sameSite: 'Lax',
  expires: Math.round(expiresAt / 1000)
}

await fs.writeFile('.auth/auth.json', JSON.stringify({ cookies: [cookieObject], origins: [] }, null, 2))
```

## Utilizzo della sessione nei test

Carica la sessione salvata nei tuoi test E2E per accedere a route protette senza login. L'esempio seguente utilizza Playwright, ma può essere adattato a qualsiasi libreria di test (ad esempio Cypress) che supporti l'iniezione di cookie.

```ts title='playwright.config.ts'
export default defineConfig({
  // ...
  projects: [
    { name: 'setup', testMatch: /global\.setup\.ts/, teardown: 'teardown' }, // [!code highlight]

    { name: 'teardown', testMatch: /global\.teardown\.ts/ }, // [!code highlight]

    {
      name: 'chromium',
      use: {
        ...devices['Desktop Chrome'],
        storageState: '.auth/auth.json' // [!code highlight]
      },
      dependencies: ['setup'] // [!code highlight]
    }
  ]
})
```
